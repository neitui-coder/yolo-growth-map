## 一、版本描述（提交审核时填）

**功能页填写：**

```
YOLO+ 是面向 YOLO+ 社群内部成员的成长档案与活动记录小程序。
核心功能：
1. 会员名录：浏览社群现届/往届成员的基本资料（姓名、公司、城市、MBTI、格言等）；
2. 个人成长档案：查看每位成员的成长时间线、参与活动记录；
3. 活动中心：浏览历届 YOLO+ 线下活动（年会、游学、区域活动）的时间、地点、亮点与参与者；
4. 社区动态：展示成员入会、晋升、生日等社群动态；
5. 个人资料编辑：成员可编辑本人的资料卡片。

本小程序为社群内部工具，不涉及任何交易、支付、收费功能，仅供成员记录与交流成长。
```

## 二、Sean 在微信公众平台后台必做（mp.weixin.qq.com）

按顺序操作，缺一不可：

### 1. 🔴 小程序备案（ICP 备案）—— 不备案无法上架
- 路径：管理后台 → 设置 → 基本设置 → 「小程序备案」
- 需准备：主体营业执照 / 法人身份证 / 法人手机号 / 备案负责人信息
- 周期：通信管理局约 1–20 个工作日，**尽早提交**
- 注：使用云开发（CloudBase）本身不额外需要域名 ICP，但小程序主体备案是强制项

### 2. 🔴 用户隐私保护指引
- 路径：管理后台 → 设置 → 服务内容声明 → 「用户隐私保护指引」
- 需声明收集的信息类型（按本小程序实际）：
  - **微信openid**（用于识别成员身份、绑定档案）
  - **会员个人资料**：姓名、所在公司、城市、生日、个人简介（成员自愿填写并展示给社群）
- 处理目的：社群成员名录与成长记录展示
- 提交后 1–2 个工作日审核

### 3. 🟡 选择服务类目
- 路径：管理后台 → 设置 → 基本设置 → 服务类目
- 建议类目：**「社交 - 社区/论坛」** 或 **「工具 - 信息查询」**
- 该类目通常无需额外资质（非经营性、无交易）

### 4. 🟡 名称与头像
- 名称：改为 `YOLO+`（基本设置 → 名称，企业号审核 1–7 天）
- 头像：上传 `assets/logo/yolo-logo-512.png`

### 5. 🟢 服务器域名（如使用了云开发外的请求）
- 本小程序主要走云开发（wx.cloud），云调用无需配域名
- `<image>` 显示外链（mmbiz.qpic.cn 活动封面）不需配域名（image 组件豁免）
- 无需额外配置（除非后续接入 wx.request 第三方接口）
