# 微信小程序发布指南

## 1. 注册流程

### 方式一：从公众号快速注册（推荐）

如果已有**已认证的服务号**，可复用公众号资质快速注册小程序：

1. 登录微信公众平台 → 小程序管理 → 快速注册并认证小程序
2. 勾选"主体资质"、"运营者信息"、"微信认证资质" → 下一步
3. 填写小程序账号信息（需要一个**未注册过公众号/小程序的邮箱**）→ 提交
4. 用邮箱收到的链接激活账号

**优势**：无需重新提交主体材料、无需对公打款
**限制**：一个公众号每月最多复用资质注册 5 个小程序
**注意**：2023年10月25日后注册的小程序，即使复用资质也需单独认证（300元/年）

### 方式二：独立注册

1. 访问 mp.weixin.qq.com → 注册 → 选择"小程序"
2. 填写邮箱（未注册过其他微信平台的）、密码
3. 选择主体类型（个人/企业）并完成验证
4. 企业需提供营业执照、对公账户等资质 + 300元认证费

### 个人 vs 企业注册区别

| 对比项 | 个人 | 企业 |
|--------|------|------|
| 认证费 | 免费 | 300元/年 |
| 微信支付 | 不支持 | 支持 |
| 获取手机号 | 不支持 | 支持 |
| 附近的小程序 | 不支持 | 支持 |
| 卡券功能 | 不支持 | 支持 |
| 服务类目 | 仅限工具、资讯、内容社区等低风险类目 | 全类目 |
| 适用场景 | 信息展示、工具类 | 商业运营、电商、服务类 |

**建议**：如果小程序涉及任何交易/支付功能，必须企业注册。纯工具/展示类可用个人注册。

## 2. 用户需要提供的信息清单

### 注册阶段
- [ ] 一个未注册过微信平台的邮箱
- [ ] 小程序管理员微信号（用于扫码绑定）
- [ ] 如走企业注册：营业执照、法人身份证、对公账户信息
- [ ] 如从公众号复用：公众号管理员微信扫码确认

### 小程序信息设置
- [ ] 小程序名称（2-30个字符，不可与已有小程序重名）
- [ ] 小程序头像（144x144px，圆形展示）
- [ ] 小程序简介（4-120个字符，清晰描述功能）
- [ ] 服务类目选择（决定功能权限，如"工具 > 信息查询"）

### 备案阶段（2024年起强制要求）
- [ ] 主体信息（个人身份证/企业营业执照）
- [ ] 负责人信息 + 手持身份证照片
- [ ] 小程序用途说明
- [ ] 服务内容标识

## 3. 代码准备清单（Claude 可以帮做的）

### 提交前必须完成
- [ ] `app.json` 配置完整：pages、window、tabBar、permission
- [ ] `project.config.json` 填写正确的 appid（注册后获得）
- [ ] 所有页面功能正常，无明显 Bug 或空白页
- [ ] 移除所有测试数据、console.log、测试账号信息
- [ ] 隐私协议声明（`__usePrivacyCheck__: true` in app.json）
- [ ] 用户隐私保护指引（在小程序后台配置）

### 代码规范检查
- [ ] 不含违规内容（赌博、色情、虚假宣传等）
- [ ] 不含诱导分享/关注行为
- [ ] 页面加载有 loading 状态，不会出现长时间白屏
- [ ] 网络请求用 HTTPS（wx.request 要求）
- [ ] 服务器域名已在小程序后台配置白名单

### 云开发相关
- [ ] 云函数部署完成
- [ ] 数据库权限规则配置正确
- [ ] 云存储文件访问权限设置合理

## 4. 提交审核步骤

### Step 1：上传代码
1. 打开**微信开发者工具**
2. 确认项目 appid 正确
3. 点击右上角「上传」按钮
4. 填写**版本号**（如 1.0.0）和**项目备注**
5. 上传成功

### Step 2：提交审核
1. 登录**微信公众平台** → 版本管理
2. 在「开发版本」中找到刚上传的版本
3. 点击「提交审核」
4. 填写信息：
   - 配置功能页面（选择主要功能页面路径）
   - 如需登录才能体验，必须提供**测试账号**
   - 确认服务类目与实际功能一致
5. 提交

### Step 3：等待审核
- 普通工具类/服务类：通常 1-2 小时（2026年已大幅提速）
- 复杂类目或首次提交：可能 1-3 个工作日
- 审核结果通过微信消息通知管理员

### Step 4：发布上线
1. 审核通过后，登录公众平台 → 版本管理
2. 在「审核版本」中点击「发布」
3. 可选择**全量发布**或**分阶段发布**（灰度）
4. 发布后用户即可搜索和使用小程序

## 5. 常见坑和注意事项

### 审核高频被拒原因

1. **类目不匹配**：选择的服务类目与小程序实际功能不一致。务必在提交前确认类目
2. **功能不完整**：存在空白页面、点击无反应、功能入口无法使用
3. **测试数据残留**：页面上有"测试"字样、假数据、Lorem ipsum
4. **诱导行为**：强制关注公众号、诱导分享才能使用功能
5. **隐私问题**：收集用户信息未说明用途、未配置隐私保护指引
6. **需登录但未提供测试号**：如果小程序必须登录才能体验核心功能，审核时必须提供测试账号
7. **简介与功能不符**：小程序简介未准确描述实际功能

### 技术注意事项

- **域名白名单**：所有 API 请求域名必须在后台「开发管理 → 服务器域名」中配置
- **HTTPS 强制**：所有网络请求必须使用 HTTPS
- **不能嵌入外部链接**：小程序中不能使用 web-view 加载未备案的域名
- **包大小限制**：单个分包不超过 2MB，总大小不超过 20MB（使用分包加载）
- **云开发配额**：免费版有调用次数和存储限制，注意不要超额

### 运营注意事项

- **备案必须**：2024年起所有小程序必须完成备案才能上架
- **年审**：认证小程序每年需要年审续费（300元）
- **不要频繁提交失败的审核**：多次被拒会延长后续审核时间
- **版本回退**：发布后如有问题，可在后台进行版本回退
- **数据统计**：发布后可在后台查看访问量、用户画像等数据

### YOLO Growth Map 小程序特别注意

- 当前项目使用微信云开发，需确保云环境已创建并正确关联
- 社区功能如涉及 UGC（用户生成内容），可能需要内容安全审核机制
- 如有用户登录功能，需配置隐私保护指引中的「微信登录」权限说明
- 建议首次提交选择「工具 > 信息查询」或「生活服务 > 生活服务」类目
