# YOLO+ 成长地图 — 产品路线图

## 用户需求和方向（2026-03-07）

### UI 优化
- 使用 Frontend Design plugin 分析并优化当前 UI
- 做样式前后对比，在小程序里加一个切换功能，点击切换不同样式方便选择
- 编辑按钮从右上角移到更合理的位置（右上角容易和其他按钮冲突）
- 图片展示太小，需要更好的排版。有图片的节点应该把图片展示得大一些
- 成长地图/时间轴 UI 需要进一步优化

### 编辑体验优化
- 标签编辑要所见即所得（WYSIWYG）：标签在哪里显示就在哪里编辑
- 整体编辑操作要更简单直观
- 减少跳转，尽量在当前页面完成编辑

### 数据存储
- 目前数据存储在 app.js 的 globalData 中（硬编码）
- 需要规划永久数据存储方案
- 已有 CloudBase 数据库设计（db-schema.json + db-seed-data.json）但未接入

### 用户激励和社交互动
- 除了成长值奖励（太理性化），需要更好玩的机制促进用户录入内容
- 加强会员和会员之间的联系
- 启发用户去录入成长地图内容的产品设计
- 探索社交互动功能
